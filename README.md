# Оптимізація типів даних: float64 → float32 / int32

## Опис завдання

Завдання: перетворити числові колонки DataFrame у менш витратні типи даних (`float32` / `int32`), порівняти використання пам'яті до і після конвертації, а також обчислити похибку середнього значення колонки `price` між `float64` і `float32`.

## Датасет

Використовується **синтетичний датасет** структурою аналогічний California Housing (20 640 рядків, 9 колонок). Генерується автоматично через `numpy` — зовнішні завантаження не потрібні.

| Колонка | Тип (до) | Тип (після) |
|---------|----------|-------------|
| `longitude` | float64 | float32 |
| `latitude` | float64 | float32 |
| `housing_median_age` | float64 | float32 |
| `total_rooms` | float64 | **int32** |
| `total_bedrooms` | float64 | **int32** |
| `population` | float64 | **int32** |
| `households` | float64 | **int32** |
| `median_income` | float64 | float32 |
| `price` | float64 | float32 |

> Колонки з цілими значеннями (`total_rooms`, `total_bedrooms`, `population`, `households`) конвертуються в `int32`, решта — у `float32`.

## Результати

### Порівняння пам'яті

| | Пам'ять |
|--|---------|
| До конвертації (float64) | **1 451.38 KB** |
| Після конвертації (float32/int32) | **725.75 KB** |
| **Економія** | **725.6 KB (50.0%)** |

Кожна колонка зменшується рівно вдвічі: `float64` займає 8 байт на значення, `float32` і `int32` — по 4 байти.

### Похибка середнього `price`

| | Значення |
|--|---------|
| Середнє (float64) | 258 454.9745642154 |
| Середнє (float32) | 258 454.9687500000 |
| Абсолютна похибка | `5.81e-03` (~0.006 $) |
| Відносна похибка | `0.00000225 %` |

Похибка практично нульова — `float32` цілком придатний для аналітичних задач.

## Висновок

Конвертація числових колонок у `float32`/`int32` дозволяє **вдвічі зменшити обсяг пам'яті** датафрейму при абсолютно незначній втраті точності (похибка < 0.000003%). Це стандартна практика оптимізації в ML-пайплайнах, особливо при роботі з великими датасетами або GPU (де `float32` є нативним типом).
